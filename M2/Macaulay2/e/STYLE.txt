Coding standards to follow when writing code in the Macaulay2 engine

Currently, files are written in a number of different standard, unfortunately.
The goal is to have a consistent format and style for the code.

Indentation:
  no tabs in the files!
  indentation for each level: 2 spaces (4 spaces is possible, but we need to be consistent).
  ** find a tool which will give us our desired formatting automatically
  (Jakob will remember the name)

  class definitions: class, {, public and private, end } all in same column
  private data is separated, placed at end of class.  Names are generally
  beginning with m, e.g. mRing.
